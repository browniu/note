# 前端架构-React-router编解码

# 项目描述

因为在使用react-router-dom的过程中发现在路由传参包含特殊符号/时，无法正确接收。因为react-router会自动对参数部分采取decodeURI（这个API无法对/转译），这个时候/后面的会被解析成路径而造成数据不完整。 这时可能会想在发送参数前对其进行encodeURIcomponent手动编码（这个API可以对包括/在内的更多特殊字符进行转译），然后在接收时使用decodeURIcomponent进行手动解码，然而react-router进行的decodeURI解码是自动的（无法阻止），所以就会对参数进行多次解码，如果参数中包含类转译字段（%2F），在接收参数时就会把%2F经过两次解码解码成/符号。

# 项目职责

  • 重构react-router的URI编码解码逻辑，解决参数中携带特殊符号的问题

# 技术亮点

  • 基于原生encodeURI之上增加了对于/[\/?#]/g等对URI分割有影响的字符进行手动转译成16进制编码。
  • 修改基础库history的编码逻辑为decodeURIComponent逻辑
  • 基于增加优于修改的思想。开发辅助处理插件，使用自定义的编码逻辑编译对特殊符号进行编解码

# 项目描述

核心逻辑的设计与实现 UMD全平台通用组件化开发

# 技术亮点

  • 使用postmessage实现iframe父子页面的跨域通信

# 前端架构-短程预加载

# 项目描述

A页面跳转B页面时，出现1-2帧的空白，查明后发现是由于资源加载导致的。 所以解决的方向是预加载B页面资源。但有别于传统的预加载，它的目标性非常明确。而且出现空白帧的原因并不是因为B页面有大型资源导致。所以可以通过touchStart与click事件的时间差进行目标明确的小资源的预加载，这个方案就实现了场景中用来优化体验的短程预加载。

# 项目指责

  • 短程预加载的逻辑设计/实现

# 技术亮点

  • 通过touchstart & click事件的时间差进行定向页面资源预加载
